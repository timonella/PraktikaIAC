# Features

### 🏢 **EventSync Manager (Центральное приложение)**
*   **Централизованное управление** — Полный контроль над мероприятиями, организациями и статусами в единой базе данных (PostgreSQL/MySQL).
*   **Мастер импорта данных** — Однократный, безопасный импорт существующих данных из Microsoft Access (.accdb) в новую, оптимизированную структуру БД.
*   **Генерация обновлений** — Создание зашифрованных дампов-обновлений для полевых команд с выборочным набором изменений.
*   **Приём и обработка отчетов** — Загрузка дампов от полевых приложений, их верификация, расшифровка и автоматическое применение изменений в центральную БД.
*   **Аналитика и отчёты** — Встроенные инструменты для анализа выполнения мероприятий (на основе Python/Pandas или .NET) с возможностью визуализации данных.

### 📱 **EventSync Field (Полевое приложение)**
*   **Полная автономность** — Работа в полевых условиях без необходимости постоянного подключения к интернету.
*   **Локальная база данных** — Использование встроенной SQLite для хранения и быстрого доступа к назначенным мероприятиям и справочной информации.
*   **Интуитивное обновление статусов** — Удобный интерфейс для оперативного изменения статусов мероприятий, добавления комментариев и фотоотчетов.
*   **Синхронизация через USB** — Простой и надежный механизм обмена данными: загрузка новых заданий и выгрузка выполненных работ через зашифрованные файлы на USB-носителе.

### 🔄 **Синхронизация и безопасность**
*   **Надёжная схема обмена** — Детально проработанный, отказоустойчивый протокол передачи данных через промежуточные носители (USB).
*   **Сквозное шифрование** — Все дампы для обмена защищены шифрованием, обеспечивающим конфиденциальность данных при физической передаче.
*   **Конфликт-менеджмент** — Алгоритмы для автоматического и ручного разрешения конфликтов данных при слиянии изменений от разных полевых команд.

### ⚙️ **Архитектура и гибкость**
*   **Чистая архитектура** — Разделение логики на классы (Class Library) для повышения сопровождаемости и тестируемости кода.
*   **Гибкий технологический стек** — Основная логика может быть реализована на **.NET 6/8** (C#) или **Python** (FastAPI), в зависимости от требований заказчика.
*   **Современный Desktop UI** — Приложения используют актуальные фреймворки: **WPF** для функционального центрального интерфейса и **WinForms/MAUI** для легковесного полевого.
*   **Независимость от Access** — После начального импорта система полностью работает на собственных СУБД, не требуя лицензий на MS Access.
